What are the wallet formats 'protobuf' and 'serialised' ?

As MultiBit has improved the file format used to store the wallets has changed over time. You normally do not have to worry about this but if you use different versions of MultiBit it is worth being aware of what they are. The file formats are:

  • serialised.1    This is the original format used and tends to produce quite large wallet files.
  • protobuf.2    Protobuf wallets were introduced in June 2012 (MultiBit version 0.4.0). They are smaller, faster to load and easier to extend. This early version of protobuf wallets could not be encrypted.
  • protobuf.3    This is an improved format of protobuf wallets that support encryption. They were introduced in MultiBit version 0.5.0.

Generally older versions of MultiBit cannot read the newer formats. When you start up MultiBit there is a conversion utility that asks you whether you want to migrate serialised wallets to protobuf. New wallets are always created in the protobuf format.

You can see which version of MultiBit you are running by choosing the 'About MultiBit' menu option. For Windows and Linux machines this is in the 'Help' menu and on Macs it is in the 'MultiBit' application menu.
