Trezor / KeepKey

The Trezor and KeepKey hardware wallets provide a way of keeping your private keys off your desktop machine thereby improving your security.

To use the Trezor or KeepKey device simply attach it using the USB cable (we recommend using a mobile phone cable for maximum reliability) and MultiBit HD will detect it automatically. After that all screens that involve private key operations (unlocking, sending, signing messages and so on) will prompt to use the device as part of the work flow.

Where possible MultiBit HD will attempt to match the screen showing on the Trezor or KeepKey so you can be sure no tampering is occurring.

Buy Trezor / KeepKey

We recommend that if you have more than 500 USD worth of bitcoin that you seriously consider purchasing one. You can start this process from within MultiBit HD by selecting this option and following the instructions.

Verify Trezor / KeepKey

This option will allow you to inspect the "Features" of your Trezor / KeepKey device so that you can tell what version it is, or the digital fingerprint of the internal firmware. Much of the information is highly technical and generally only needed if you are experiencing problems.

Wipe Trezor / KeepKey

This option will allow you to wipe a Trezor / KeepKey back to its factory settings.

Make sure you have a paper copy of your "wallet words" for the Trezor / KeepKey before wiping!

Once a Trezor / KeepKey device is wiped there is no going back to the earlier wallet without the "wallet words". If you find yourself needing to do this MultiBit HD does support Trezor / KeepKey wallet words as part of the Restore process. See the related articles for more details.

The next time the Trezor / KeepKey is attached MultiBit HD will prompt you to initialise the device with a fresh set of "wallet words" generated from the Trezor / KeepKey itself. These are never revealed to MultiBit HD and remain private to the Trezor / KeepKey device.

My Trezor / KeepKey is very slow to respond

Sometimes if MultiBit HD is not shut down properly a connection can be held open to the Trezor / KeepKey resulting in long delays during communication. The quickest way to resolve this is to simply detach the device and then reattach it. After that it should work as normal.
